## 💡 關鍵設計

### 統一帳號系統
- 使用 `oneUsers` 和 `oneCurrentUser`
- 各系統保持獨立（互不影響）
- 員工管理自動同步到所有系統

### 數據隔離
- 入口系統：`oneUsers`, `oneCurrentUser`
- 報價系統：`users`, `currentUser`（同步自入口）
- 倉儲系統：獨立數據庫
- 打掃系統：獨立數據庫

### 權限控制
- **主管**：可訪問所有系統（包括員工管理）
- **員工**：可訪問報價、倉儲、打掃系統
- **會計**：只能查看報價系統

## ⚠️ 注意事項

1. **系統獨立性**
   - 各系統維護時互不影響
   - 只共用帳號系統
   - 倉儲和打掃系統需獨立部署

2. **帳號同步**
   - 員工管理會自動同步到報價系統
   - 倉儲和打掃系統需手動配置帳號（或未來整合）

3. **本地測試**
   - 倉儲系統：需啟動 `node server.js`（port 3001）
   - 打掃系統：需啟動 `node server.js`（port 3000）
   - 報價系統：直接開啟 HTML（GitHub Pages）
